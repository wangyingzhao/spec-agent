## 任务
为以下 Spec 生成任务清单。

Spec 目录：$ARGUMENTS

## 执行步骤

### Step 1: 前置检查
确认以下文件存在：
- `$ARGUMENTS/spec.md`
- `$ARGUMENTS/plan.md`
- `$ARGUMENTS/contracts/`

如果 plan.md 不存在，提示用户先执行 `/sdd:plan $ARGUMENTS`（召唤猫头鹰）。

### Step 2: 读取 spec-kit 原生模板
读取 `.specify/templates/commands/tasks.md`，理解 spec-kit 对 tasks 产物的格式要求。

### Step 3: 生成任务清单
严格按照 spec-kit tasks 模板的格式，读取 plan.md、spec.md、contracts/，
在 `$ARGUMENTS/` 下生成 `tasks.md`，要求：
- 每个 Task 关联 spec.md 中的 FR 和 AC 编号
- 标注依赖关系
- 标注可并行的 Task `[P]`
- 精确到文件路径

### Step 4: 输出摘要
展示 tasks.md 内容概要，告知用户审阅后执行 `/sdd:implement`。
